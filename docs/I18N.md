# Internationalization (i18n) Setup

This project uses `i18next` and `react-i18next` for internationalization support.

## Features

- **Supported Languages**: English (default) and Portuguese
- **Auto-detection**: Automatically detects browser language
- **Persistence**: Language preference is stored in localStorage
- **Easy switching**: Language switcher component in the header
- **Dynamic translations**: Country names and content classes adapt to selected language

## Structure

```
src/
├── config/
│   └── i18n.ts           # i18n configuration
├── locales/
│   ├── en.json           # English translations
│   └── pt.json           # Portuguese translations
├── hooks/
│   └── useTranslatedData.ts  # Custom hook for translated data
└── components/
    └── LanguageSwitcher.tsx  # Language toggle component
```

## Usage

### In Components

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t("app.title")}</h1>
      <p>{t("app.subtitle")}</p>
    </div>
  );
}
```

### Using the Custom Hook

For country names and content classes:

```tsx
import { useTranslatedData } from "../hooks/useTranslatedData";

function MyComponent() {
  const { getCountryName, getContentClassName } = useTranslatedData();

  return (
    <div>
      <h2>{getCountryName("BR")}</h2>
      <p>{getContentClassName("finance")}</p>
    </div>
  );
}
```

### Changing Language Programmatically

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { i18n } = useTranslation();

  const changeLanguage = (lng: string) => {
    i18n.changeLanguage(lng);
  };

  return (
    <button onClick={() => changeLanguage("pt")}>Switch to Portuguese</button>
  );
}
```

## Data Structure

### Mock Data

All mock data now uses language-neutral identifiers:

- **Country names**: Use country codes (e.g., 'BR', 'US') instead of hardcoded names
- **Content classes**: Use category IDs (e.g., 'finance', 'health') instead of hardcoded names

This ensures that all labels automatically adapt to the selected language through the translation system.

## Configuration

The i18n configuration is in `src/config/i18n.ts`:

- **Default language**: English (`en`)
- **Fallback language**: English (`en`)
- **Detection order**: localStorage → browser navigator
- **Cache**: localStorage

## Language Switcher

The language switcher is included in the application header and shows:

- EN / PT buttons
- Highlights the active language
- Tooltips for accessibility
- Persists selection to localStorage
